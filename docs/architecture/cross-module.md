# Связи между доменами

Как домены ссылаются друг на друга в модульном монолите.

См. также: [Onion](onion.md), [Events](events.md), [CQRS](cqrs.md).

---

## Правила

- Домен не импортирует `Domains\Other\Application\...` или `...\Infrastructure\...` в свой код.
- Для реакции на изменение в другом домене — слушатель на [доменное событие](events.md), зарегистрированный в домене-подписчике.
- Чтение чужих данных в рамках одного запроса — через `QueryBus` (`ask(...)`), не через прямой вызов чужого репозитория.
- Ссылка на запись другого домена хранится как `id`; домен не тянет чужие репозитории.

---

## Чего пока нет

Port + Adapter, синхронные cross-domain write-транзакции и Outbox **не реализованы** — текущий объём (CRUD per-domain) их не требует. Если появится операция, атомарно меняющая несколько доменов, решение оформляется отдельным ADR (`docs/adr/`) до реализации.
